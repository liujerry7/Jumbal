# LIFE.md — Jumbal's life-planning memory

Jumbal's durable, cross-session memory of the user's life-planning: **goals, tasks, ideas,
thoughts, reminders.** This is personal/life context — *not* product orchestration (that lives in
each product repo's `JUMBAL.md`). Read this at the start of a life-planning session so you continue
with continuity, not a cold start.

Owner: the user (liujerry7@gmail.com). Started 2026-06-19.

## How this file works

- **Goals** — outcomes the user is working toward; give each a rough horizon and the *why*.
- **Active tasks / TODOs** — concrete next actions. `[ ]` open / `[x]` done; date them; note any
  due date.
- **Ideas** — sparks worth exploring (possible projects, app features, experiments). Not yet
  committed.
- **Thoughts & principles** — reflections, values, and preferences that should shape the advice
  Jumbal gives.
- **Reminders** — time-bound items; surface them as their date nears.
- **Done / log** — recently completed items, kept briefly for context, then pruned.

**When the user sends a raw brain-dump:** parse each item into the right bucket, dedupe against
what's already here, date it, link related items (e.g. a task under its goal), and flag anything
ambiguous back to the user rather than guessing. Keep this file current — promote, complete, and
prune as life moves. If a bucket grows large, split it into its own file under a
`life-planning/` folder and leave a pointer here.
